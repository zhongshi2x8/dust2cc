# SteamDT Inline Panel Design

## Goal

修复 `steamdt` 页面使用错误价格做分析的问题，并恢复网页内 AI 面板：

- 页面内 AI 面板插入到商品信息区的空白区域
- K 线图内只显示简洁重点位标记
- 完整重点位报告放在网页内 AI 面板
- `side panel` 继续作为主视图

## Root Cause Summary

- `steamdt` 的当前价格来源不稳定，当前实现会过早采用错误的 DOM 或 payload 价格候选，导致页面显示价格和分析使用价格不一致。
- 为了避免图表布局被打乱，之前临时关闭了 `steamdt` 的网页内 AI 面板和图表标记，导致页面体验回退。

## Design

### Data

- 引入统一的价格候选排序逻辑。
- 优先选择与最新 K 线收盘价接近、且语义上更像“当前成交/最低在售价”的价格。
- 当 DOM / payload 候选价格明显离谱时，回退到最新 K 线收盘价，避免 `¥23` 这种误判进入分析结果。

### Web Inline Panel

- `steamdt` 恢复网页内 AI 面板。
- 面板采用紧凑摘要卡样式，挂载在商品详情上方空白区，而不是 K 线图下方。
- 页面内面板默认展示本地分析摘要、关键价位、重点位报告；不承担完整深度分析职责。

### Chart Markers

- `steamdt` 图内仅保留简洁重点位标记。
- 不再启用大面积覆盖式原生注释层，避免再次打乱图表布局。

### Side Panel

- `side panel` 保持完整分析视图。
- 页面内卡片和 `side panel` 共用同一份纠正后的价格与信号结果。

## Validation

- 自动测试覆盖价格候选选择逻辑。
- 构建通过。
- 重点人工验证：
  - `steamdt` 页面价格是否与页面主价格一致或接近
  - 网页内 AI 面板是否出现在上方空白区
  - K 线图是否仍保持正常布局
